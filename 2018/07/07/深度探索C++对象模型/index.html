chapter1. 关于对象

###c++对象模型
可能在布局上引起额外负担的操作由virtual引起包括：
virtual function
virtual base class

  • 简单对象模型
    feature：低效、空间复杂度高、降低编译器设计复杂度

    每一个成员对应一个指针，成员本身不存在object中，object中存放的是pointers。

    被应用于“指向成员的指针”概念

  • 表格驱动对象模型
    成员信息分放在data member table 和member function table中，class object本身存放指向两个表格的指针

    被应用于虚函数表

  • C++对象模型
    非静态数据成员放在每一个class object中，静态数据成员放在class之外
    静态与非静态函数成员都放在class object之外

    虚函数支持的步骤：
    1.每一个class产生一堆指向virtual functions的指针，放在表格中，即virtual table
    2.每一个class object被添加一个指针，指向相关的virtual table，指针被称为vptr，vptr由构造函数、复制构造函数设定重置，每一个class关联的type_info object（支持RTTI使用）也由vb指出，放在表格的第一个slot中

优缺点：空间和存取效率高，仅仅修改非静态数据成员也要重新编译

C++内存布局是按照块内声明顺序进行内存排列，块之间顺序是不确定的
